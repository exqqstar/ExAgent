# Module Notes

Module pages are organized by understanding boundary, not strictly by source folder.

Use this template when adding or expanding a module page:

```md
# Module Name

## Responsibility

What this module owns and what it does not own.

## State

State this module owns or mutates.

## File Map

Files covered by this module page and their roles.

## Key Flows

Important functions in execution order.

## Connections

Upstream callers and downstream dependencies.

## Extension Points

How this module usually changes.
```
