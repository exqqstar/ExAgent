# Benchmark Notes

Benchmark notes capture architecture feedback from real evaluation runs.

These files are not leaderboard claims. They are working notes for:

- benchmark harness behavior,
- runtime failure modes,
- token and tool-call economics,
- follow-up optimization candidates.

Use this folder when a benchmark run exposes a recurring system issue that
does not fit a single module page yet.
