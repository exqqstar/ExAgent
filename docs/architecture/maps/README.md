# Architecture Maps

Maps answer where things are and how to navigate the codebase.

- [module-map.md](module-map.md): logical modules and their relationships.
- [file-map.md](file-map.md): source files grouped by responsibility.
- [state-map.md](state-map.md): runtime and durable state ownership.

Maps should stay short. Put detailed explanations in `modules/` or `flows/`.
